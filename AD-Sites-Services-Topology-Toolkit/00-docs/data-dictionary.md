# Active Directory Sites And Services Diagram Data Dictionary

This dictionary summarizes the raw discovery files and normalized diagram inputs. The low-level requirements remain the source of truth for behavior and validation rules.

## Raw Discovery Exports

| File | Purpose | Key Fields |
| --- | --- | --- |
| `forest.csv` | Forest-level context. | `ForestName`, `RootDomain`, `ForestMode`, `SchemaMaster`, `DomainNamingMaster`, `Domains`, `GlobalCatalogs`, `SiteCount`, `Notes` |
| `domains.csv` | Domain-level context and FSMO roles. | `DomainName`, `NetBIOSName`, `DomainMode`, `PDCEmulator`, `RIDMaster`, `InfrastructureMaster`, `Notes` |
| `sites.csv` | AD replication sites. | `SiteName`, `DistinguishedName`, `Description`, `Location`, `Options`, `WhenCreated`, `WhenChanged`, `SourceServer`, `Notes` |
| `subnets.csv` | AD replication subnets and site assignments. | `SubnetName`, `Cidr`, `SiteName`, `Location`, `Description`, `DistinguishedName`, `WhenCreated`, `WhenChanged`, `Notes` |
| `site-links.csv` | AD site links. | `SiteLinkName`, `Transport`, `SitesIncluded`, `SiteCount`, `Cost`, `ReplicationFrequencyInMinutes`, `Schedule`, `Options`, `DistinguishedName`, `Notes` |
| `site-link-bridges.csv` | Explicit site link bridge objects when present. | `BridgeName`, `Transport`, `SiteLinksIncluded`, `DistinguishedName`, `Notes` |
| `ip-transport.csv` | Transport-level settings, especially bridge-all-site-links. | `TransportName`, `BridgeAllSiteLinks`, `DistinguishedName`, `Options`, `Notes` |
| `domain-controllers.csv` | Domain controllers and site placement. | `HostName`, `Name`, `Domain`, `Forest`, `SiteName`, `IPv4Address`, `OperatingSystem`, `IsGlobalCatalog`, `IsReadOnly`, `OperationMasterRoles`, `PreferredBridgeheadTransports`, `Notes` |
| `replication-connections.csv` | Optional configured replication connection objects. | `ConnectionId`, `ConnectionName`, `SourceServer`, `DestinationServer`, `SourceSite`, `DestinationSite`, `Transport`, `Enabled`, `AutoGenerated`, `Schedule`, `Options`, `Notes` |
| `replication-partner-metadata.csv` | Optional observed replication partner metadata. | `MetadataId`, `Direction`, `SourceServer`, `DestinationServer`, `NamingContext`, `LastSuccess`, `LastFailure`, `ConsecutiveFailureCount`, `ResultCode`, `ResultMessage`, `Status`, `Notes` |
| `replication-failures.csv` | Optional observed replication failures. | `FailureId`, `SourceServer`, `DestinationServer`, `NamingContext`, `FirstFailure`, `LastFailure`, `ConsecutiveFailureCount`, `ResultCode`, `ResultMessage`, `Status`, `Notes` |
| `replication-topology-edges.csv` | Optional configured and observed replication topology evidence. | `ReplicationEdgeId`, `EvidenceType`, `SourceServer`, `DestinationServer`, `NamingContext`, `LastSuccess`, `LastFailure`, `ResultCode`, `Status`, `EvidenceId`, `Notes` |
| `replication-health-summary.csv` | Optional per-domain-controller replication health summary. | `DomainController`, `SiteName`, `PartnerMetadataCount`, `ConfiguredConnectionCount`, `FailureCount`, `QueueOperationCount`, `LastSuccess`, `LastFailure`, `Status`, `Notes` |
| `run-metadata.json` | Discovery run context, warnings, and safe parameter echo. | Timestamp, user, computer, PowerShell version, AD module version, credential flag and username only, warnings, errors |
| `raw-inventory.json` | Full structured raw inventory. | All raw datasets and schedule/detail fields that do not fit cleanly in CSV |

## Normalized Diagram Inputs

| File | Purpose | Notes |
| --- | --- | --- |
| `diagram-objects.csv` | AD sites rendered as diagram nodes. | `ObjectType` should be `ADSite` for MVP. |
| `line-of-sight-links.csv` | Deduplicated unordered site pairs derived from AD site links. | One row per site pair, with all contributing site links preserved in `Notes` and `transform-summary.json`. |
| `ports-protocols.csv` | Firewall review rows generated from a configurable AD port profile. | Treat as review input, not proof that every port is required everywhere. |
| `ad-site-domain-controller-expansion.csv` | Domain controllers by AD site. | Preferred AD Sites expansion table. |
| `ad-site-subnets.csv` | Subnets assigned to AD sites. | Subnet assignments are discovered AD configuration and may be stale. |
| `ad-domain-server-expansion.csv` | Compatibility expansion file for the sample renderer pattern. | Temporary compatibility output; table labels must not imply these are AD domains in AD Sites mode. |
| `replication-connections.csv` | Configured replication connection objects for review. | These are configured objects, not observed health evidence. |
| `replication-partner-metadata.csv` | Observed replication partner metadata for review. | These rows show read-only observed state where AD exposes it. |
| `replication-failures.csv` | Observed replication failures for review. | These rows are evidence only and do not remediate AD. |
| `replication-topology-edges.csv` | Renderer-ready replication topology evidence. | `EvidenceType` distinguishes `ConfiguredConnection`, `ObservedPartnerMetadata`, `ReplicationFailure`, and `ReplicationQueue`. |
| `replication-health-summary.csv` | Per-DC replication health summary. | Summarizes configured, observed, failure, and queue counts. |

## Important Interpretation Rules

- AD site links do not prove active replication connections. KCC bridgehead selection can change.
- Configured replication connection objects and observed replication partner metadata are different evidence types and must not be merged without preserving `EvidenceType`.
- Replication health data is read-only evidence. The toolkit does not remediate AD and does not prove network reachability.
- Multi-site site links must be expanded, deduplicated by unordered site pair, and clearly labeled as derived topology.
- The IP transport `BridgeAllSiteLinks` setting changes how site-link-derived reachability should be interpreted.
- AD subnet object names normally are CIDR values, so `SubnetName` and `Cidr` should match unless normalization found a malformed or special case.
- Full AD schedule data belongs in JSON; CSV should contain a readable summary such as `24x7` or `Custom - see raw-inventory.json`.
- Anonymized outputs still may reveal environment shape. The anonymization map must be protected.
